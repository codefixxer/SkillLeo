# Professional Icon Component System

## Overview
This icon system provides a comprehensive, modern, and professional set of outlined and solid icons suitable for enterprise applications. All icons are consistent in weight, size, and style, following design principles from LinkedIn, Microsoft, and Atlassian.

## Features
- **24 Professional Icons** covering all common use cases
- **Outlined & Solid variants** for each icon
- **5 Size options** (xs, sm, md, lg, xl)
- **8 Color variants** including semantic colors
- **Interactive states** with hover effects
- **Accessibility features** with focus states
- **Dark mode support**
- **Responsive design**

## Component Props

```blade
<x-ui.icon 
    name="home"           // Icon name (required)
    variant="outlined"    // outlined|solid (default: outlined)
    size="md"            // xs|sm|md|lg|xl (default: md)
    color="primary"      // default|primary|secondary|success|warning|danger|light|dark
    stroke-width="1.5"   // SVG stroke width for outlined icons
    class="custom-class" // Additional CSS classes
/>
```

## Quick Usage

### Basic Icons
```blade
<x-ui.icon name="home" />
<x-ui.icon name="notification" variant="solid" color="primary" />
<x-ui.icon name="heart" size="lg" color="danger" />
```

### Interactive Buttons
```blade
<button class="icon-btn primary">
    <x-ui.icon name="add" color="light" />
</button>

<button class="icon-text bg-blue-600 text-white px-4 py-2 rounded-lg">
    <x-ui.icon name="save" color="light" size="sm" />
    <span>Save Changes</span>
</button>
```

### Navigation
```blade
<nav class="flex gap-4">
    <a href="#" class="icon-text hover:text-blue-600">
        <x-ui.icon name="home" class="hover-lift" />
        <span>Home</span>
    </a>
</nav>
```

## Adding New Icons

To add a new icon to the system:

1. **Find the SVG**: Use Heroicons (heroicons.com) for consistency
2. **Add to component**: Edit the icon component's switch statement
3. **Follow the pattern**:

```blade
@case('new-icon-name')
    @if($variant === 'solid')
        <svg fill="currentColor" viewBox="0 0 24 24">
            <!-- Solid version SVG path -->
        </svg>
    @else
        <svg fill="none" stroke="currentColor" viewBox="0 0 24 24" stroke-width="{{ $strokeWidth }}">
            <!-- Outlined version SVG path -->
        </svg>
    @endif
    @break
```

## Best Practices

### Icon Selection
- Use outlined icons for most cases
- Use solid icons for active states or emphasis
- Keep icon choice consistent across similar actions

### Size Guidelines
- `xs` (12px): Small inline icons, badges
- `sm` (16px): Form inputs, small buttons
- `md` (20px): Default size, navigation
- `lg` (24px): Primary actions, headers
- `xl` (32px): Hero sections, large displays

### Color Usage
- `default/secondary`: General content
- `primary`: Primary actions, links
- `success`: Confirmations, positive states
- `warning`: Alerts, attention needed
- `danger`: Errors, destructive actions
- `light`: Icons on dark backgrounds
- `dark`: High contrast situations

### Accessibility
- Always include meaningful alt text context
- Use semantic colors appropriately
- Ensure sufficient color contrast
- Provide keyboard navigation for interactive icons

## Performance Considerations

- Icons are inline SVG for optimal performance
- No external dependencies or icon fonts
- Minimal CSS footprint
- GPU-accelerated animations
- Optimized for tree-shaking

## Browser Support

- All modern browsers (Chrome, Firefox, Safari, Edge)
- IE11+ (with basic functionality)
- Mobile browsers (iOS Safari, Chrome Mobile)
- Graceful degradation for older browsers
